# Block And Character Device Catalog

This page documents the default AT386 system configuration generated by the modern staging and `idconfig` replacement path, not the reduced boot-floppy profile. The inventory below is derived from the staged `mdevice`, `sdevice`, and `node` metadata that the current build produces.

The generated metadata contains active configured devices plus source-backed staged records from the historical tree. Some networking records still have historical `sdevice` state `N`, but the default build spec overrides the network and TLI modules on so they appear in the generated switch tables. A staged `node` record is still not enough by itself; the target driver or module also has to be enabled.

!!! note
    This is a catalog of the active default switch-table entries plus selected staged source metadata that matters to the documented kernel interfaces. It does not attempt to list every historical add-on driver under `uts/add-ons/`.

## How To Read The Tables

- `Major` is the configured switch-table major range.
- `Units` is the sum of configured units across the staged `sdev` records.
- `Mask` and `Type` use the meanings described in [Kernel Device Drivers](kernel-drivers.md).
- `Static node metadata` shows representative lines from staged `node` files when this tree declares them. An empty field means the default configuration does not stage a `node` file for that driver.

## Source-Backed Clone Node Metadata

Several user-visible endpoint names are declared through the `clone` major. Their staged `node` files bind the public name to the clone driver, and clone-open dispatches to the real STREAMS character-driver target in `cdevsw[]` when that target is configured.

The network and TLI targets below are source-backed and have staged `node` metadata. Their historical AT386 `sdev` records remain disabled, but the default generated kernel enables these modules explicitly; the reduced boot-floppy profile excludes them.

| --- | --- | --- | --- |
| [`arp`](drivers/arp.md) | `clone arp c arp` | Enabled; historical `sdev` is `N` | Source-backed network endpoint metadata. |
| [`icmp`](drivers/icmp.md) | `clone icmp c icmp` | Enabled; historical `sdev` is `N` | Source-backed network endpoint metadata. |

| [`rawip`](drivers/rawip.md) | `clone rawip c rawip` | Enabled; historical `sdev` is `N` | Source-backed raw IP endpoint metadata. |
| [`ticlts`](drivers/ticlts.md) | `clone ticlts c ticlts` | Enabled; historical `sdev` is `N` | Source-backed TLI endpoint metadata. |
| [`ticots`](drivers/ticots.md) | `clone ticots c ticots` | Enabled; historical `sdev` is `N` | Source-backed TLI endpoint metadata. |

Those names matter because the public `/dev` path, clone target, and target minor are all part of the ABI once the target is configured. If a userspace ABI relies on a specific path, the `node` file is the right place to document or preserve it.

## Scope Limits

- This page reflects the default full AT386 configuration. The smaller boot-floppy profile excludes many of these drivers.
- The source tree contains additional driver code under `uts/add-ons/`, but those add-ons are not part of the default generated configuration tables yet.
- An empty `Static node metadata` field does not mean the driver is unusable; it only means this `master.d` metadata does not stage a `node` file for it.

For individual driver notes, see the pages under [drivers/](drivers/index.md).
